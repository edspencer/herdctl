- **Exposure:** All herdctl deployments using gRPC communication
- **Fix Available:** Yes - update to protobufjs 7.5.5+
- **Status:** Not fixed despite 32 days of awareness

### 2. esbuild RCE (Discovered Friday) 🆕

- **Exposure:** All builds, CI/CD pipelines
- **Fix Available:** Yes - update to esbuild 0.28.1+
- **Status:** Not fixed

**What This Means:** Attackers can compromise herdctl either at runtime (via malicious protobuf data) OR during the build process (via supply chain attack). Both vectors are currently unprotected.
