#### 1. Path Traversal Fix for Windows (31c675c) - **MITIGATION STRENGTHENED** ✅

**PR:** #210
**Date:** 2026-03-17
**Files Changed:**
- `packages/core/src/state/utils/path-safety.ts` (CRITICAL HOT SPOT)
- `packages/core/src/state/utils/__tests__/path-safety.test.ts`
- `.changeset/fix-windows-path-sep.md`

**What Changed:**

Fixed `buildSafeFilePath()` to use `path.sep` instead of hardcoded `/` for cross-platform compatibility:

```diff
// Before (broken on Windows):
- if (!resolvedPath.startsWith(`${resolvedBase}/`) && resolvedPath !== resolvedBase) {
+ const basePrefix = resolvedBase.endsWith(sep) ? resolvedBase : `${resolvedBase}${sep}`;
+ if (!resolvedPath.startsWith(basePrefix) && resolvedPath !== resolvedBase) {
    throw new PathTraversalError(baseDir, identifier, resolvedPath);
```

**Why It Matters:**

On Windows, `path.resolve()` returns paths with `\` separators. The original hardcoded `/` check caused **every state file operation** to throw `PathTraversalError`, making herdctl completely unusable on Windows.

**Security Impact:** 🟢 **POSITIVE**

This fix **strengthens** Finding #001 (Path Traversal - FIXED) by:
- Making the mitigation work correctly on Windows (was broken, throwing false positives)
- Preventing double-separator edge cases (`//` or `\\\\`) that could bypass validation
- Adding explicit test coverage for cross-platform behavior

**Previous State:**
- ✅ POSIX: Path traversal mitigation working correctly
- ❌ Windows: False positives on ALL operations (herdctl unusable)
- ⚠️ Root paths: Double-separator could fail validation incorrectly

**Current State:**
- ✅ POSIX: Path traversal mitigation working correctly
- ✅ Windows: Path traversal mitigation working correctly
- ✅ Root paths: Correct handling of paths that already end with separator

**Test Coverage Added:**


- ✅ Automatic cleanup after processing
- ✅ Docker path translation (prevents path confusion)

**Code Review:**

**Recommendation:** Consider adding magic byte validation using `file-type` library for defense-in-depth (LOW priority).

**Risk Level:** MEDIUM (multiple layers of validation, but new untrusted input)
