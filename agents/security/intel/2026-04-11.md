#### 2. ⚠️ Discord File Attachment Handling (Commit 3f947a0 - NEW ATTACK SURFACE)
**Files:**
- `packages/discord/src/manager.ts`
- `packages/core/src/config/schema.ts`
- `packages/discord/src/file-sender-mcp.ts` (new)

**Date:** 2026-03-09

**Feature:** File upload/download for Discord agents

**Security Controls Implemented:**
1. ✅ **MIME type whitelist** (`["image/*", "application/pdf", "text/*"]` default)
2. ✅ **File size limits** (10 MB default, configurable)
3. ✅ **Max files per message** (5 default)
4. ✅ **Path traversal protection** - rejects `..` and absolute paths
5. ✅ **UUID-based subdirectories** - prevents filename collisions
6. ✅ **Download timeouts** (30 seconds)
7. ✅ **Automatic cleanup** (delete after processing)
8. ✅ **Docker path translation** - handles host→container mapping

**Download Path Pattern:**
```
{workingDir}/{download_dir}/{messageUUID}/{attachmentId}-{filename}
```

**Validation:**
```typescript
.refine((v) => !v.includes("..") && !v.startsWith("/"), {
  message: "download_dir must be a relative path without '..' segments",
})
```

**Risk Assessment:**

| Risk | Severity | Mitigation | Status |
|------|----------|------------|--------|
| Path traversal via download_dir | HIGH | Zod validation rejects `..` and `/` | ✅ Mitigated |
| Malicious file upload | MEDIUM | MIME type whitelist + size limits | ✅ Mitigated |
| Prompt injection via text files | MEDIUM | Limited to configured MIME types | ⚠️ Residual risk |
| Resource exhaustion | LOW | Size limits + max files + timeouts | ✅ Mitigated |
| Filename collisions | LOW | UUID subdirectories + attachment IDs | ✅ Mitigated |

**Concerns:**
- ⚠️ **Prompt injection:** Text files are inlined directly into agent prompts
  - Mitigation: Only configured MIME types accepted
  - Recommendation: Consider content scanning/sanitization
- ⚠️ **Third-party content:** Files fetched from Discord CDN without additional verification
  - Relies on Discord's infrastructure security
  - No malware/virus scanning implemented

**Recommendation:** Consider adding:
1. Rate limiting on file uploads per user/channel
2. Virus/malware scanning for uploaded files
3. Content sanitization for inlined text
4. Audit logging for file operations

**Overall Assessment:** **Acceptable risk** - Controls are comprehensive for stated use case
